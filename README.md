# Vibe Ad Studio

A polished, interactive AI ad-generation tool. Paste a URL в†’ we scrape the page,
generate 3вЂ“5 on-brand ad creatives (headline, body, CTA, score, image, tone),
and let you inline-edit, regenerate, shorten, change tone, or make any ad bolder.

> Inspired by Vibe Studio. Built with **Next.js (App Router) + Tailwind + OpenAI + Cheerio**.

---

## Features

- **URL в†’ Ads**: Cheerio-based scraper extracts title, meta description, and the first 3вЂ“5 key paragraphs.
- **Generates 3вЂ“5 ads**, each with:
  - Headline В· Body В· CTA
  - **Score (80вЂ“100)** with circular meter
  - **Tone** (Bold, Friendly, Playful, Premium, Minimal, Urgent, Inspiring)
  - AI-style image (Picsum, deterministic seed вЂ” no extra API key needed)
- **Inline editing** of headline, body, and CTA (contentEditable, with caret management).
- **Per-ad actions**: Regenerate В· Shorten В· Make bolder В· Change tone (dropdown).
- **AI loading sequence** with multi-stage status, animated progress bar, and skeleton previews.
- **Works without an OpenAI key** вЂ” falls back to a deterministic generator for demos.
- Clean SaaS aesthetic: soft shadows, glassy cards, gradient text, subtle animations.

## Quick start

```bash
# 1. Install
npm install

# 2. (optional) configure OpenAI
cp .env.example .env.local
#    then edit .env.local and set OPENAI_API_KEY=sk-...

# 3. Run
npm run dev
# в†’ http://localhost:3000
```

If `OPENAI_API_KEY` is not set, the app runs in **Demo mode** вЂ” ads are generated by a built-in deterministic copywriter so the entire flow still works end-to-end.

## API

All routes live under `/api` and use the Node runtime.

### `POST /api/scrape`

Body:
```json
{ "url": "https://example.com" }
```
Returns:
```json
{
  "scraped": {
    "url": "...",
    "siteName": "...",
    "title": "...",
    "description": "...",
    "ogImage": "...",
    "paragraphs": ["..."]
  }
}
```

### `POST /api/generate`

Body:
```json
{ "scraped": { /* output of /api/scrape */ }, "count": 4 }
```
Returns:
```json
{
  "ads": [
    {
      "id": "abc123",
      "headline": "...",
      "body": "...",
      "cta": "...",
      "tone": "Bold",
      "score": 92,
      "image": "https://picsum.photos/seed/.../800/520"
    }
  ],
  "usedAI": true
}
```

### `POST /api/edit`

Body:
```json
{
  "ad":     { "id": "...", "headline": "...", "body": "...", "cta": "...", "tone": "Bold" },
  "action": "regenerate" | "shorten" | "bolder" | "tone",
  "tone":   "Playful",
  "scraped": { /* used as context */ }
}
```
Returns:
```json
{ "ad": { /* updated ad */ }, "usedAI": true }
```

## Deployment

### Vercel (recommended for now)

1. Push this folder to a Git repo.
2. Import the repo on [Vercel](https://vercel.com/new).
3. Add the `OPENAI_API_KEY` env var in **Project Settings в†’ Environment Variables**.
4. Deploy. That's it.

### Future: Oracle Cloud Infrastructure (OCI)

The codebase is intentionally portable. To migrate later:

- **Containerize**: build a small Docker image (Next.js standalone output or `next start`).
- **Compute**: deploy to OCI Container Instances or an OKE (Kubernetes) cluster.
- **Secrets**: store `OPENAI_API_KEY` in OCI Vault and inject as env vars.
- **Edge / CDN**: front the service with OCI Web Application Firewall + CDN.
- **Object storage** (when DB/assets are added later): use OCI Object Storage.

Nothing in this app is Vercel-specific вЂ” there are no Vercel functions, edge runtimes, or image-loader lock-ins. The API routes use the standard Node.js runtime.
